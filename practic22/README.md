## 1. Цель
Реализация системы балансировки нагрузки для Node.js приложения с использованием Nginx и HAProxy.

## 2. Реализация

### Компоненты:
| Сервис | Порт | Назначение |
|--------|------|------------|
| Backend #1 | 3000 | Основной сервер |
| Backend #2 | 3001 | Основной сервер |
| Backend #3 | 3002 | Резервный сервер |
| Nginx | 8080 | Балансировщик #1 |
| HAProxy | 8081 | Балансировщик #2 (альтернатива) |
| Frontend | 3003 | Клиентское приложение |

### Алгоритмы балансировки:
- **Round Robin** (по умолчанию) — запросы распределяются по кругу
- **Backup server** — резервный сервер используется только при недоступности основных

### Отказоустойчивость (Nginx):
```nginx
server 127.0.0.1:3000 max_fails=2 fail_timeout=30s;
```
- `max_fails=2`: сервер считается недоступным после 2 неудачных попыток
- `fail_timeout=30s`: сервер исключается из ротации на 30 секунд

## 3. Инструкция по запуску

### Запуск backend-серверов:
```bash
# Терминал 1
PORT=3000 node server.js

# Терминал 2  
PORT=3001 node server.js

# Терминал 3 (резервный)
PORT=3002 node server.js
```

### Запуск Nginx:
```bash
cd backend
sudo nginx -c $(pwd)/nginx_load_balancer.conf
```

### Запуск HAProxy:
```bash
haproxy -f backend/haproxy.cfg
```

### Запуск frontend:
```bash
cd frontend
PORT=3003 npm start
```

## 4. Тестирование

### Проверка распределения запросов:
```bash
curl http://localhost:8080/api/products
# Повторить несколько раз — ответы должны чередоваться между серверами 3000 и 3001
```

### Проверка отказоустойчивости:
1. Остановить сервер на порту 3000
2. Выполнить запрос — все запросы идут на 3001
3. Остановить сервер на порту 3001 — запросы идут на резервный 3002
