## 1. Описание реализации

В рамках данного практического занятия была проведена полная миграция серверной части приложения с файлового хранилища (`db.json`) на реляционную СУБД **PostgreSQL**. Архитектура приложения была рефакторирована для соблюдения принципов **SOLID** и **ООП**.

### Ключевые изменения:

1.  **Внедрение PostgreSQL**:
    *   Данные пользователей, товаров и refresh-токенов теперь хранятся в таблицах БД.
    *   Используется драйвер `pg` (node-postgres) для выполнения SQL-запросов.
    *   Реализована автоматическая инициализация схемы БД при первом запуске (`initDb.js`).

2.  **Архитектурный рефакторинг (SOLID & OOP)**:
    *   **SRP (Single Responsibility Principle)**: Логика работы с данными вынесена из контроллеров в отдельные классы-репозитории (`UserRepository`, `ProductRepository`, `TokenRepository`). Контроллеры отвечают только за обработку HTTP-запросов и ответов.
    *   **DIP (Dependency Inversion Principle)**: Контроллеры зависят от абстракций (интерфейсов репозиториев), а не от конкретных реализаций SQL-запросов. Это позволяет легко заменять источник данных (например, на MongoDB) без изменения бизнес-логики.
    *   **Инкапсуляция**: Подключение к базе данных инкапсулировано в классе `Database` (Singleton). Генерация уникальных ID и хеширование паролей скрыты внутри слоев сервиса/репозитория.
    *   **Адаптация данных**: Реализован маппинг полей из формата БД (`snake_case`, например `image_url`) в формат API (`camelCase`, например `imageUrl`) внутри репозиториев.

3.  **Безопасность и производительность**:
    *   Использование параметризованных SQL-запросов для защиты от SQL-инъекций.
    *   Хранение секретов (DB URL, JWT Secrets) в переменных окружения (`.env`).
    *   Использование пула соединений (`Pool`) для эффективной работы с БД.

## 3. Инструкция по запуску

### Предварительные требования
1.  Установленный **Node.js** (v16+).
2.  Установленный и запущенный **PostgreSQL** (локально или облачный сервис, например ElephantSQL).
3.  Менеджер пакетов `npm`.

### Шаг 1: Настройка базы данных
1.  Создайте новую базу данных в PostgreSQL с именем `tea_shop_db`.
2.  Запомните пароль пользователя `postgres` (или создайте нового пользователя).

### Шаг 2: Настройка Backend
1.  Перейдите в папку `backend`:
    ```bash
    cd backend
    ```
2.  Установите зависимости:
    ```bash
    npm install
    ```
3.  Создайте файл `.env` в корне папки `backend` и заполните его:
    ```env
    PORT=3000
    CLIENT_URL=http://localhost:3001
    ACCESS_SECRET=your_access_secret
    REFRESH_SECRET=your_refresh_secret
    # Замените password на ваш пароль от PostgreSQL
    DATABASE_URL=postgres://postgres:password@localhost:5432/tea_shop_db
    ```
4.  Запустите сервер:
    ```bash
    node server.js
    ```
    *При первом запуске скрипт `initDb.js` автоматически создаст таблицы `users`, `products`, `refresh_tokens` и добавит администратора.*

### Шаг 3: Настройка Frontend
1.  Откройте новый терминал и перейдите в папку `frontend`:
    ```bash
    cd frontend
    ```
2.  Установите зависимости и запустите клиент:
    ```bash
    npm install
    npm start
    ```
3.  Приложение будет доступно по адресу `http://localhost:3001`.
